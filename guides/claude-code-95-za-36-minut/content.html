Что такое Claude Code

Claude Code — это ИИ-агент от Anthropic, который работает прямо на вашем компьютере. Это не просто чат-бот — это полноценный помощник-разработчик, способный создавать сайты, Telegram-ботов, автоматизации и многое другое.

Ключевое отличие

Claude Code — это агентный режим Claude. Он может читать и редактировать файлы, запускать терминальные команды, работать с Git, искать по кодовой базе — всё в автоматическом режиме.

Даже если вы не программист — вайб-кодинг (vibe coding) делает AI-разработку доступной каждому. Claude Code помогает стать в 10-15 раз эффективнее.

Установка Claude Code

Самый простой способ — установка через VS Code как расширение.

Через VS Code

  1. Откройте VS Code
  2. Перейдите в раздел Extensions (Ctrl+Shift+X)
  3. Найдите «Claude Code» от Anthropic
  4. Нажмите Install
  5. Авторизуйтесь через аккаунт Anthropic

Через терминал (CLI)

# Установка через npm
npm install -g @anthropic-ai/claude-code

# Запуск
claude
💡

Совет: Claude Code также доступен как десктоп-приложение (Mac/Windows), веб-приложение (claude.ai/code) и расширения для JetBrains IDE.

04 06:34

Интерфейс расширения

Интерфейс Claude Code в VS Code состоит из панели чата, встроенного терминала и индикатора текущего режима работы.

Основные элементы

Панель чата — основное пространство для диалога с Claude. Вы вводите запросы, Claude отвечает и показывает свои действия.

Терминал — Claude может выполнять команды прямо в терминале VS Code, устанавливать пакеты, запускать скрипты.

Индикатор режима — показывает текущий режим (Ask, Auto и т.д.) и используемую модель.

Файловый контекст — Claude видит открытые файлы и может читать/редактировать любые файлы в проекте.

05 08:41

Режимы работы

Claude Code имеет четыре режима работы, каждый с разным уровнем автономности:

💬

Ask

Только отвечает на вопросы. Не может менять файлы или запускать команды. Безопасный режим для изучения кода.

⚡

Auto

Основной рабочий режим. Claude читает, пишет код, запускает команды — но запрашивает подтверждение для рискованных действий.

📋

Plan Mode

Claude анализирует задачу и составляет пошаговый план без выполнения. Идеален для сложных проектов.

🔒

Bypass Permissions

Полная автономия без запросов подтверждения. Используйте осторожно — Claude выполняет все действия сразу.

⚠️

Bypass Permissions — мощный, но опасный режим. Claude сможет удалять файлы, запускать произвольные команды и делать git push без подтверждения. Используйте только когда уверены в том, что делаете.

06 11:08

Effort (усердность мышления) и токены

Effort — уровень «глубины размышления» Claude перед ответом. Влияет на качество ответа и расход токенов.

УровеньОписаниеКогда использовать
LowБыстрые, короткие ответыПростые вопросы, мелкие правки
MediumСбалансированный режимТипичные задачи разработки
HighГлубокий анализ перед ответомСложная архитектура, дебаг
Что такое токены?

Токены — единицы текста (~0.75 слова). Каждое сообщение расходует входные токены (ваш запрос + контекст) и выходные (ответ Claude). Чем длиннее контекст и ответ — тем больше расход.

07 12:40

Лимиты подписок

Каждая подписка имеет дневные/месячные лимиты на количество токенов. При достижении лимита скорость ответов снижается или доступ временно ограничивается.

Как не сжигать лимиты

• Используйте /compact для сжатия контекста

• Выбирайте правильную модель — Haiku дешевле для простых задач

• Не отправляйте огромные файлы целиком в чат

• Пишите точные, конкретные запросы

09 15:14

Контекстное окно

Контекстное окно — объём информации, которую Claude «помнит» в рамках одного диалога. Это один из самых важных концептов для эффективной работы.

Контекстное окно Claude Code ┌──────────────────────────────────────────┐ │ Системный промпт (CLAUDE.md, режим) │ │ История диалога (ваши сообщения) │ │ Прочитанные файлы (код проекта) │ │ Результаты команд (вывод терминала) │ │──────────────────────────────────────────┤ │ Рекомендуемый предел: 150-200K токенов │ │ Максимум модели: до 1M токенов │ └──────────────────────────────────────────┘
Почему не стоит превышать 150-200K токенов

При большом контексте Claude начинает «терять фокус» — может забывать ранние инструкции, путать файлы или давать менее точные ответы. Используйте /compact для сжатия контекста.

10 15:52

CLAUDE.md — системный промпт

CLAUDE.md — это файл с инструкциями, который Claude читает в начале каждого диалога. По сути, это ваш «системный промпт» для настройки поведения агента.

Уровни CLAUDE.md

РасположениеОбластьОписание
~/.claude/CLAUDE.mdГлобальныйДля всех проектов — ваш стиль, язык, предпочтения
./CLAUDE.mdПроектныйВ корне проекта — стек, архитектура, правила
./src/CLAUDE.mdПапкаВ подпапке — специфичные инструкции для части проекта

Что писать в CLAUDE.md

# Пример проектного CLAUDE.md

## Проект
Это Telegram-бот для учёта расходов на Python + aiogram.

## Стек
- Python 3.12, aiogram 3.x
- PostgreSQL + asyncpg
- Docker Compose для деплоя

## Правила
- Пиши на русском
- Используй type hints
- Логируй через loguru
- Тесты: pytest + pytest-asyncio

## Структура
src/
  handlers/   # Обработчики команд
  services/   # Бизнес-логика
  models/     # Модели данных
  db/         # Работа с БД
💡

Совет: CLAUDE.md — самый мощный инструмент настройки. Хорошо написанный CLAUDE.md радикально повышает качество ответов Claude.

11 19:22

Команды /init и /compact

/init — инициализация проекта

Команда /init анализирует ваш проект и автоматически создаёт CLAUDE.md с описанием стека, структуры и правил. Отличная отправная точка для нового проекта.

/compact — сжатие контекста

Когда контекстное окно разрастается, /compact суммаризирует предыдущий диалог, сохраняя ключевую информацию и освобождая место. Используйте регулярно в длинных сессиях!

# Инициализация нового проекта
/init

# Сжатие контекста (можно с инструкцией)
/compact
/compact сохрани информацию о структуре БД
12 20:55

Папка .claude

Claude Code хранит свои настройки и данные в двух местах:

Глобальная папка ~/.claude/ ├── CLAUDE.md # Глобальный системный промпт ├── settings.json # Настройки (хуки, разрешения) ├── keybindings.json # Кастомные горячие клавиши ├── projects/ # Память по проектам │ └── my-project/ │ └── memory/ # Автопамять проекта └── plugins/ # Установленные плагины/скиллы Локальная папка ./.claude/ (в корне проекта) ├── settings.json # Настройки проекта └── skills/ # Скиллы проекта
💡

Совет: Добавьте .claude/ в .gitignore, если не хотите шарить настройки с командой. Или наоборот — коммитьте, чтобы вся команда использовала одни скиллы и правила.

13 24:30

Скиллы

Скиллы — это готовые наборы инструкций, которые автоматизируют рутинные задачи. Скиллы вызываются через слэш-команды.

Как работают скиллы

Скилл — это Markdown-файл с frontmatter и инструкциями. Когда вы вызываете /skill-name, Claude загружает содержимое скилла и следует его инструкциям.

# Пример скилла: .claude/skills/deploy.md
---
name: deploy
description: Деплой проекта на сервер
---

## Шаги деплоя
1. Запусти тесты: `pytest`
2. Собери Docker-образ: `docker build -t app .`
3. Запуши на сервер: `docker compose up -d`
4. Проверь логи: `docker logs app`

Существуют также маркетплейс-скиллы от сообщества — готовые решения для TDD, дебага, код-ревью, frontend-разработки и многого другого.

14 27:55

Субагенты

Субагенты — это дочерние экземпляры Claude, которые основной агент запускает для параллельного выполнения задач.

Главный агент (ваш диалог) │ ├─→ Субагент 1: исследование кодовой базы │ ├─→ Субагент 2: поиск документации │ └─→ Субагент 3: выполнение тестов
Когда Claude использует субагентов

• Параллельное исследование нескольких файлов или концепций

• Сложные задачи, требующие разделения на подзадачи

• Защита основного контекстного окна от переполнения

💡

Совет: Субагенты расходуют дополнительные токены, но экономят ваш основной контекст. Для масштабных проектов это часто выгоднее.

15 29:37

MCP — Model Context Protocol

MCP — протокол, позволяющий Claude подключаться к внешним сервисам и инструментам: базам данных, API, браузеру, файловым системам и многому другому.

🌐

Браузер

Playwright — Claude может открывать сайты, кликать, заполнять формы, делать скриншоты.

📄

Notion

Создание и редактирование страниц, баз данных, комментариев в Notion.

🎨

Figma

Чтение дизайнов, конвертация макетов в код, создание диаграмм.

🚀

Vercel

Деплой проектов, просмотр логов, управление доменами.

MCP-серверы настраиваются в settings.json и могут быть глобальными или проектными.

16 31:23

Хуки

Хуки — это shell-команды, которые автоматически выполняются в ответ на события Claude Code (вызов инструмента, отправка сообщения и т.д.).

// Пример в settings.json
{
  "hooks": {
    "on-tool-call": [
      {
        "matcher": "Edit",
        "command": "echo 'Файл отредактирован'"
      }
    ]
  }
}
Типы хуков

• on-tool-call — перед выполнением инструмента

• on-tool-result — после выполнения инструмента

• on-prompt-submit — при отправке сообщения

17 32:06

Автопамять и /dream

Автопамять — Claude запоминает важную информацию между сессиями: ваши предпочтения, решения, контекст проекта. Память хранится в файлах в папке ~/.claude/projects/.

/dream — улучшение CLAUDE.md

Команда /dream запускает анализ проекта и накопленного опыта, затем предлагает улучшения для CLAUDE.md. По сути, Claude «спит» и «видит сны» о том, как стать полезнее в вашем проекте.

💡

Совет: Запускайте /dream после значимых рабочих сессий — Claude проанализирует, что работало хорошо, и обновит инструкции.

18 33:21

Экономия контекста

Эффективное использование контекстного окна — ключ к продуктивной работе с Claude Code:

Правила экономии

1. Используйте /compact регулярно — не ждите, пока контекст переполнится.

2. Пишите точные запросы — «Исправь ошибку в файле X, строка Y» вместо «что-то не работает».

3. Разбивайте задачи — одна задача на диалог лучше, чем мега-сессия на 5 часов.

4. Выбирайте правильную модель — Haiku для мелочей, Sonnet для основной работы, Opus для сложного.

5. Хороший CLAUDE.md — чёткие инструкции экономят токены на уточняющих вопросах.

6. Не вставляйте большие файлы в чат — пусть Claude читает их сам через инструменты.
