1. Что вообще такое SEO и почему это до сих пор работает

SEO — про то, как попасть наверх Google по нужному запросу. Распределение кликов жестокое:

  • 40% кликов уходят на 1-ю позицию
  • 20% — на 2-ю
  • Дальше — резкое падение

Четыре оси работы:

  1. Поиск ключей — не все запросы одинаково ценны
  2. On-page SEO — что и как написано на странице
  3. Technical SEO — как сайт собран (скорость, Core Web Vitals)
  4. Off-page SEO — что говорят про вас другие сайты

AI SEO ≈ обычное SEO. Когда ChatGPT, Perplexity или Google AI отвечает на вопрос — он лезет в поиск, забирает топ и пересказывает. Хорошо ранжируешься в Google → попадаешь в LLM-ответы.

2. Сетап: Antigravity + Claude Code + CLAUDE.md

Качаем antigravity.google — это бесплатный десктоп от Google поверх VS Code. В таб Extensions ставим Claude Code.

  1. Создаём пустую папку проекта (например, seo-brief)
  2. Кладём в корень файл CLAUDE.md с инструкциями и правилами проекта
  3. Открываем папку в Antigravity, запускаем Claude и даём промпт на сборку сайта

Промпт для первой итерации:

Build a beautiful website with three pages:
home, blog index, service index. Clone the design
from the attached screenshot. Use Static Site
Generation. Stack: Next.js with SSG output.

Скриншот для референса бери с dribbble.com — без него получится AI-slop. Открой первый подходящий лендинг по теме, сделай скрин, прикрепи к промпту.

Static, Server, Client — почему это критично

Три типа рендеринга, как в пиццерии:

  • SSG (Static Site Generation) — пицца уже готова, отдают за 10 секунд. Google заходит — страница есть. Это то, что нам нужно
  • SSR (Server-Side) — пиццу готовят при заказе, надо ждать. Google ждёт, иногда уходит
  • CSR (Client-Side) — делайте пиццу сами. Google вообще не увидит контент

Без SSG все остальные усилия по SEO бесполезны — если краулер не видит страницу, ранжировать нечего.

3. Поиск выигрышных ключевых слов

Просить Claude «дай 20 ключей по сантехнике» — путь в никуда. Модель не знает ни volume, ни difficulty. Нужен внешний инструмент. Автор использует SEMrush (есть бесплатный триал).

Фильтры в Keyword Magic Tool

Вбиваешь корневой ключ, например plumber. Получаешь миллион вариаций. Оставляешь только то, что реально берётся:

  • Keyword Difficulty ≤ 30 — даёт реальный шанс пробиться
  • Volume ≥ 100 — меньше нет смысла оптимизировать
  • Intent: Informational — для блога. Для сервисных страниц — Commercial

Отсекай ключи, которые на самом деле чужие бренды (например, «Hydrojet Plumbing» — название компании). Их ты не отранжируешь, и попытка только сожжёт ресурсы.

Дополнительные источники

  • Questions tab в SEMrush — готовые темы для блога: «How much does a plumber cost?»
  • Adjacent topics — то, что читают до готовности купить: «best water softener 2026», «signs you need a new water heater»
  • Competitor research — вбиваешь домен конкурента, забираешь его рангирующиеся ключи

Собранные ключи экспортируешь как CSV и кладёшь в проект как keywords.csv.

4. Тактика №1: блог-посты на потоке

Шаг 1 — первый сырой пост

Получи Pexels API key, положи в .env:

PEXELS_API_KEY=your_key_here

Промпт:

Create a blog post for the keyword "plumber for low
water pressure". Build a keyword cluster — pull related
keywords from keywords.csv or generate them. Pull
images from Pexels using the API key in .env.

Что такое keyword cluster

Один пост ранжируется не за один ключ, а за десятки. Корневой ключ «how to unclog a drain» обрастает кластером:

  • unclog kitchen sink
  • unclog bathroom sink
  • drain won't drain
  • slow drain sink
  • home remedy clogged drain
  • what dissolves hair in drain

Все они вшиваются в одну страницу. На выходе — пост, который ловит трафик с десятков смежных запросов.

Шаг 2 — добавить голос, юмор, истории

Сырой результат всегда AI-slop: «in today's fast-paced world, maintaining a functional household...». Это никто не дочитает, и Google это видит по dwell time.

Создаём в проекте папку references/ с пятью файлами:

  • voice.md — как ты пишешь
  • humor.md — конкретные приёмы юмора, dad-jokes, side notes
  • opinions.md — твои сильные позиции
  • stats.md — реальные цифры бизнеса (сколько лет на рынке, сколько прочисток сделали)
  • stories.md — анекдоты с реальных вызовов

Чем заполнить: транскрипты звонков, посты в LinkedIn, переписка в Gmail, любые тексты, где звучит твой голос. Скармливаешь Claude — он сам разложит по файлам:

Update voice.md, humor.md, opinions.md, stats.md and
stories.md based on these LinkedIn posts:
[вставляешь тексты]

Если своих текстов нет — найди референс. Автор в видео цитирует tone «I've got 99 problems but humor ain't one» и просит Claude использовать как humor reference. Этого достаточно, чтобы получить пост с настоящими шутками вместо корпоративного бубнежа.

Шаг 3 — украсть формулу из топа Google

SEO — это «чёрный ящик»: 200+ ранжирующих сигналов, точные веса не известны никому. Самый прямой путь — скопировать то, что уже работает.

В промпте для переписывания поста:

Before rewriting, search Google for the primary
keyword. Take the top 3 non-Reddit results.
Analyze each: word count, H2/H3 structure,
number of images, sub-topics covered.
Average those metrics — that's the winning formula.
Use the average word count, H2 count and structure
for our post. Then inject our voice, humor, stats
and stories from references/.

На выходе — пост размером и структурой как у топ-3, но в твоём голосе. Например, вместо «in today's fast-paced world» получаешь:

«Your toilet has been hissing for longer than most celebrity marriages. It's 11 p.m., you're Googling 'running toilet' instead of sleeping, and you're starting to take the sound personally.»

5. Тактика №2: сервисные страницы (zipper-схема)

Блог даёт topical authority — Google видит, что ты эксперт в нише. Сервисные страницы конвертят этот трафик в деньги, потому что они под коммерческие запросы.

Schema «zipper»

У бизнеса есть две оси:

  • Услуги: emergency plumber, drain cleaning, hot water repair, leak detection
  • Локации: Vancouver, Toronto, Montreal, Edmonton

Сшиваешь их попарно в страницы:

emergency-plumber-vancouver
emergency-plumber-toronto
drain-cleaning-vancouver
drain-cleaning-toronto
...

Не делай тысячи таких страниц. Контент похож, и Google пометит это как doorway pages. Пара десятков — нормально. Сотни — рискованно.

Поиск денежных ключей

В SEMrush сортируй по Cost Per Click по убыванию. Дорогие клики = реальные конкуренты платят за этот запрос Google Ads = это коммерческий запрос. Паттерн в выдаче будет «service + city» — это и есть твой шаблон сервисной страницы.

Шаблон страницы — копия с конверсии

Не лепи каждую сервисную страницу по отдельному дизайну. Запусти Google Ads на 5–10 вариантов лендинга, найди тот, у кого конверсия 15–20%, и используй его шаблон для всех сервисных страниц.

6. On-page SEO: 80+ сигналов в один промпт

Нет смысла учить вручную, что такое meta description, alt-теги, schema markup. Один раз кладёшь в проект onpage-checklist.md с 80 пунктами и просишь Claude прогнать через него каждый пост.

Что должно быть в чек-листе (минимум):

  • Ровно один <h1>; чёткая иерархия H2/H3
  • Primary keyword в первых 100 словах
  • 2–3 external links на авторитетные источники
  • 3–5 internal links на свои блог-посты и сервисные страницы
  • 4–8 раскрытых вопросов в посте (для FAQ-разметки)
  • Meta title (≤ 60 символов) и meta description (≤ 160)
  • Open Graph и Twitter Card метатеги
  • Alt-текст для всех изображений с целевыми ключами
  • Schema.org разметка (Article, FAQPage, HowTo)
  • Slug в kebab-case с ключевым словом

Промпт для прогона:

Update the blog post at [path] using all items from
onpage-checklist.md. Maintain the existing voice,
humor and personality — do not flatten the writing.

7. Technical SEO: sitemap, robots, Lighthouse 100/100

Три обязательных артефакта:

  1. sitemap.xml — список всех страниц для Google
  2. robots.txt — что разрешено краулить (закрой только админки)
  3. Lighthouse score 100/100 по всем четырём осям: SEO, Accessibility, Best Practices, Performance

Прогон Lighthouse

В Chrome: F12 → DevTools → таб Lighthouse → Analyze page load. Получаешь четыре score-а и список конкретных проблем (LCP request discovery, render-blocking requests, legacy JavaScript и т.д.).

Копируешь весь отчёт целиком и в Claude:

Here's my Lighthouse report — fix everything to get
100% on performance, accessibility, best practices and
SEO. Also generate sitemap.xml and robots.txt.

[paste the full report]

Если первая попытка дала, скажем, 87% — копируй новый отчёт обратно в Claude и итерируешь, пока не будет 100/100.

8. Обернуть всё в Claude Code Skill

Когда пайплайн отлажен — нет смысла повторять промпты руками. Создаёшь скилл, который запускается одной командой /blog.

Промпт на создание скилла:

Create a Claude Code skill called "blog". Steps:
1. Pick a keyword from keywords.csv that hasn't been used
2. Build a keyword cluster
3. Search Google for the keyword, analyze top 3
4. Generate the post using references/voice|humor|stats|opinions|stories
5. Pull images from Pexels via .env API key
6. Apply onpage-checklist.md
7. Use the existing technically-optimized template
8. Add to sitemap.xml

После этого новая сессия → команда /blog → готовый пост.

Каденция важна. Не публикуй 1000 постов за день — Google ловит спайки и помечает домен как спам. Реалистичный график:

  • Дни 1–7: по одному посту в день
  • Неделя 2: по 2 в день
  • Неделя 3+: 3–4 в день максимум

9. Off-page SEO: четыре безопасных способа

Авторская позиция: off-page — самая опасная часть SEO. Каждый апдейт алгоритма Google в первую очередь убивает тех, кто покупал бэклинки. Но есть четыре легитимных способа.

① Broken Backlink Swap

В SEMrush делаешь Site Audit конкурента (например, Homestars). Находишь его внешние битые ссылки (404). Пишешь владельцу страницы, на которой стоит битая: «У вас сломанная ссылка, у меня есть страница на ту же тему — поменяйте».

② Guest Posting

В Google ищешь:

"plumbing" + "write for us"

Получаешь сайты, которые принимают гостевые статьи с обратной ссылкой на тебя.

③ HARO / Hero Journalism

Журналисты ищут экспертов. Подписываешься на сервисы типа HARO, отвечаешь на вопросы по теме («как прочистить унитаз без вантуза»), получаешь упоминание в статье + бэклинк.

④ Платные качественные бэклинки

Есть авторитетные домены с pay-to-play форматом. Платишь $100–$1000, получаешь публикацию с обратной ссылкой. Только реальные домены — не PBN-фермы.

Чего НЕ делать: покупать «100 бэклинков за $5». Это PBN (Private Blog Networks) — фермы из 10 000 мусорных сайтов. Google детектит за месяцы и накладывает санкции, которые лечатся годами или вообще не лечатся. Дешёвый «SEO-эксперт» с Upwork в 90% случаев привезёт тебе именно это.

10. Деплой: GitHub → Vercel → Search Console

Деплой за 60 секунд

  1. Создай приватный репозиторий на GitHub
  2. В Claude Code: «push the whole project to this repo: [URL]»
  3. Зайди на vercel.com, подключи GitHub-аккаунт
  4. Import Project → выбери репо → Framework Preset: Next.js → Deploy

Готово. Можно подключить кастомный домен через Vercel Domains или импортировать с Namecheap.

Четыре шага после деплоя

  1. Google My Business — самый низко висящий фрукт. Бесплатное listing-окно справа от поиска по «plumber Toronto». Тысячи кликов в месяц при нулевых усилиях
  2. Google Search Console — добавляешь URL prefix, верифицируешь через HTML meta-тег (Claude вставит за секунду), сабмитишь sitemap.xml
  3. Request Indexing — в Search Console в строке поиска вбиваешь URL новой страницы → Request Indexing. Вместо ожидания недель страница в индексе через сутки. Лимит ~10 запросов в день
  4. Google Analytics — отслеживание поведения и конверсий, чтобы знать, какой пост приносит лидов

11. Главная мысль про Google

SEO — чёрный ящик. 200+ сигналов, никто не знает точные веса. Но логика Google проста: они зарабатывают на рекламе. Чтобы показывать рекламу — нужны лояльные пользователи. Чтобы пользователи возвращались — нужно показывать им качественный контент.

Из этого вытекает главное правило: сначала контент, потом метрики. Если пост не дочитывают — никакая идеальная on-page оптимизация не вытянет его в топ. Если пост залипает — Google сам понесёт его наверх. Юмор, голос, истории, реальные цифры из бизнеса — это не «приятный бонус», а ключевой ранжирующий сигнал через dwell time, scroll depth, return visits.

🔑 Ключевые мысли

  1. Static Site Generation — не опция. Без неё Google либо не видит страницу, либо ранжирует ниже. Это первое, что должно быть в CLAUDE.md
  2. Не все ключи равны. Difficulty ≤ 30, volume ≥ 100, informational intent — три фильтра, которые отсеивают 99% мусора
  3. Голос важнее структуры. AI-slop читать невозможно, dwell time в ноль, ранжироваться нечем. Voice/humor/stats files — обязательная часть проекта
  4. Кради формулу у топ-3. SEO — это reverse engineering того, что Google уже отранжировал. Средние метрики топ-3 — это и есть «рабочий ответ»
  5. Сервисные страницы конвертят, блог даёт authority. Без блога сервисные не поднимутся, без сервисных — некуда конвертить трафик
  6. Off-page — последняя задача, не первая. PBN убивают сайты, и большинство «SEO-экспертов» с биржи делают именно их. Лучше потратить время на ещё 50 блог-постов
