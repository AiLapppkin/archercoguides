1. Понять проблему, которую решает система

Claude страдает амнезией. При переполненном контекстном окне он начинает уверенно галлюцинировать — и это самое опасное: ошибки звучат как факты.

В обычной работе это случается в ~1% случаев. Но когда контекста в чате много — шанс растёт до 10%. Практический пример: провёл трёхчасовой созвон с клиентом, договорился о деталях — на следующий день Клод не помнит ничего. Потеря контекста = потеря клиента.

Решение Карпати: дать Claude права на чтение и запись в локальную папку. Он сам создаёт там структуру, пишет заметки, организует их. Папка работает параллельно с контекстным окном — и остаётся между сессиями.

💡 Чем это отличается от классического RAG

Обычный RAG (день сурка): загружаешь PDF → задаёшь вопрос → Клод отвечает → забывает. Завтра повторяй с нуля.

Метод Карпати (снежный ком): каждый новый источник укрепляет базу. Знания накапливаются. Одна статья — 15 обновлённых страниц за один проход.

2. Скачать и открыть Obsidian

  1. Зайди на официальный сайт obsidian.md и скачай приложение под свою систему.
  2. Установи и открой. Папку хранилища пока не создавай — пусть Claude сделает это сам.
  3. В настройках Obsidian зайди в раздел Файлы и ссылки и укажи папку по умолчанию для вложений (например, изображения). Это нужно, чтобы все картинки из Web Clipper лежали аккуратно в одном месте.
  4. Включи тёмную тему, если хочется: Настройки → Оформление → Тёмная тема.

3. Запустить Claude Code и отправить промт Карпати

  1. Открой Claude Code (Antigravity или терминал).
  2. Создай новую папку проекта — например, Jarvis. Это имя твоей системы памяти.
  3. Запусти Claude Code: нажми Extensions → выбери расширение → Ctrl+L для закрытия правого окна.
  4. Напиши Клоду стартовую команду: 
    Привет. Я хочу, чтобы ты создал структуру папок по образцу, который я тебе сейчас кину.
Если есть вопросы — задавай, но в целом просто следуй инструкции,
создай каркас моей базы знаний.
[вставь сюда промт Карпати с GitHub]
  5. Полный промт Карпати есть в описании к видео по первой ссылке. Вставляй как есть.

Claude создаст файлы INDEX.md, LOCK.md и базовую структуру папок.

Структура папок

Вся система держится на трёх компонентах:

  • raw/ — сырые исходники: статьи, PDF, транскрипты YouTube. Файлы сюда кладутся и не трогаются.
  • wiki/ — заметки, которые Claude пишет сам на основе исходников. Концепты, профили, идеи, связи.
  • CLAUDE.md — системный файл с правилами. Объясняет Клоду, как вести эту базу. Превращает его из чатбота в дисциплинированного wiki-редактора.

4. Открыть папку в Obsidian

  1. Вернись в Obsidian.
  2. Нажми Открыть папку и выбери папку, которую создал Claude (например, Jarvis).
  3. Открой Граф в левой панели.

Ты увидишь сеть узлов — визуальную карту знаний. Узлы — это заметки, стрелки — связи между ними. Чем больше материалов загружаешь, тем гуще паутина.

💡 Два окна, одна папка

Держи открытыми два окна одновременно: Claude Code (пишет и читает файлы) и Obsidian (показывает ту же папку визуально). Это не два разных проекта — это один и тот же проект с двух точек зрения.

5. Добавить первый материал

Напиши Клоду в новом чате, открыв папку проекта:

Я хочу сделать Вики по [твоя тема]. Назовём её Jarvis — мой персональный ассистент,
который знает меня и помогает принимать лучшие решения.
Я буду кидать тебе статьи, кейсы, идеи, инструменты.
Сначала вот мой первый URL. Пройдись по нему, собери весь контент и картинки,
разложи по папкам raw/source и сделай первую wiki-страницу.
[вставь ссылку]

Claude прочитает статью, скачает изображения, создаст wiki-страницы с концептами и перелинкует их между собой.

Три операции с базой

  1. Добавить материал — закидываешь URL или файл, Claude интегрирует в базу.
  2. Задать вопрос — открываешь папку в Claude Code, пишешь запрос. Клод просматривает wiki, переходит по ссылкам, выдаёт ответ с указанием файлов-источников.
  3. Линтинг — раз в пару недель просишь Claude проверить базу: найти битые ссылки, противоречия, сомнительные утверждения и навести порядок.

6. Подключить Obsidian Web Clipper

  1. Открой интернет-магазин Chrome (или своего браузера).
  2. Найди расширение Obsidian Web Clipper и установи.
  3. Закрепи иконку на панели расширений.
  4. Зайди в настройки клиппера и пропиши путь сохранения — например, raw/web-clip в твоём хранилище Jarvis.

Теперь на любой статье нажимаешь иконку — клиппер сам вытягивает текст и сохраняет в нужную папку. Claude увидит новый файл при следующем запросе.

7. Понять ограничения классического Obsidian RAG

⚠️ Три проблемы, о которых молчат обзорщики

  1. Раздутый INDEX.md. Одно описание файла весит ~75 токенов. При 100 файлах — 7 500 токенов на каждый запрос. При 10 000 файлах — 750 000 токенов. Claude вынужден перечитывать весь индекс при каждом обращении. Это прямые деньги из бюджета.
  2. Нет семантического поиска. Система ищет по темам и ключевым словам, а не по смыслу. Синонимы и контекст она не понимает.
  3. Не рассчитана на большие объёмы. Obsidian RAG — магия на 100 заметках. На 10 000 — разоришься и сойдёшь с ума.

Правило: Obsidian — для того, что постоянно меняется (идеи, черновики, активные проекты). Для статичных архивов (транскрипты, архивы переписок, книги) — нужен Pinecone.

8. Выстроить полную архитектуру памяти

Правильная система состоит из четырёх компонентов, которые не конкурируют, а дополняют друг друга:

📄 CLAUDE.md

Личность и правила. Досье сотрудника: имя, роль, жёсткие инструкции. Клод читает этот файл первым делом. Никогда не меняется.

🔷 Obsidian

Мастерская. Активная мыслительная работа: черновики, заметки, текущие проекты. Связи между идеями. Здесь происходит анализ.

🗄️ Pinecone

Склад. Гигантский архив статичных данных: транскрипты, книги, архивы писем. Векторизирует текст без участия Claude — в сотни раз дешевле.

📓 Notebook LM

Глубокое погружение. Бесплатный инструмент от Google. До 300 источников за раз — для задач, которые обычный Claude не потянет.

Когда использовать Pinecone вместо Obsidian

Если у тебя есть двухчасовой транскрипт вебинара — не загружай в Obsidian. Запусти скрипт векторизации: он переведёт текст в числа и положит в Pinecone. Тяжёлый Claude в этом процессе вообще не участвует.

При следующем запросе Pinecone за миллисекунду найдёт нужный абзац среди миллионов строк и отдаст только его. Чтобы подключить Pinecone к Claude, просто напиши в чате: «Подключись к моему Pinecone» — и он проведёт по шагам.

Пример использования Notebook LM

Задача: исследовать нишу инвестиций в AI-стартапы. Скажи Клоду:

Зайди в Notebook LM, создай блокнот, залей туда 200 ресурсов по теме
инвестиций в AI-стартапы и сделай мне выжимку.

Выводы сохрани в Obsidian или Pinecone для дальнейшего использования.

🔑 Ключевые мысли

  1. Одна база — один проект. Не вали всё в кучу. Один Jarvis для YouTube-канала, другой для клиентских кейсов, третий для инструментов. Каждый знает свою тему глубоко.
  2. Obsidian — это про логику, Pinecone — про память. Obsidian мыслит вместе с тобой, выстраивая связи. Pinecone запоминает факты. Это разные инструменты для разных задач.
  3. Не строй UI поверх базы. У тебя уже есть идеальный интерфейс — это Claude. Просто открывай папку проекта и общайся в терминале.
  4. Один раз собрал — работает всю жизнь. Каждая новая статья делает базу дороже. Система не стартует, она только растёт.
  5. Масштабируй заранее. Начинаешь с Obsidian, но строй с прицелом на Pinecone. Когда дорастёшь до 1000+ файлов, переводи архив на векторный поиск — не жди, пока разоришься на токенах.
