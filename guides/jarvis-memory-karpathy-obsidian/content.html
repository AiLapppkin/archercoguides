Собрать Jarvis: долгосрочная память Claude Code на Obsidian

Настроить самообновляющуюся базу знаний по методу Андрея Карпати. Claude Code сам пишет и связывает заметки — ты открываешь Obsidian и видишь, что он запомнил.

◉ автор · Мастадонт ◉ основа · @karpathy Obsidian-RAG ◉ видео на YouTube ↗

💡 О чём этот гайд

Видео объясняет систему Obsidian-RAG от Андрея Карпати: Claude Code ведёт локальную «Википедию» в папке Markdown-файлов — сам создаёт структуру, пишет заметки, связывает сущности и концепты. Ты получаешь ИИ-ассистента, который не забывает контекст и становится умнее с каждым диалогом. Видео собрало 18M просмотров за неделю.

✅ Что ты освоишь

  1. Как развернуть Jarvis-базу с нуля за 30 минут в связке Claude Code + Obsidian + Antigravity.
  2. Как скормить первую статью и получить автоматически 10–15 взаимосвязанных wiki-страниц.
  3. Как сохранять веб-страницы одной кнопкой через Obsidian Web Clipper в свой vault.
  4. Как задавать Jarvis составные вопросы и получать синтез со ссылками на свои же источники.
  5. Как понять потолок системы и когда включать Pinecone или NotebookLM вместо Obsidian.

01 ▸ Установи Obsidian и Antigravity

Скачай obsidian.md под свою ОС и запусти. В настройках (⚙ внизу слева) → Appearance → включи Dark Mode. Окно оставь открытым — папку подключим позже.

Скачай Antigravity — бесплатный IDE от Google в preview-доступе. Он нужен как среда, где живёт Claude Code и терминал рядом с файлами.

Совет на потом
В Settings → Files & Links укажи default attachment folder raw/assets/ — изображения со статей будут складываться туда.
Два окна
Работаешь одновременно в двух окнах: Antigravity (Claude пишет файлы) и Obsidian (визуальный просмотр + граф).

02 ▸ Создай папку Jarvis и запусти Claude Code

  1. Создай пустую папку и назови её Jarvis (или своим ником).
  2. В Antigravity: File → Open Folder → выбери папку.
  3. Открой терминал: Terminal → New Terminal (или Ctrl + `).
  4. Введи команду claude и нажми Enter.
  5. При первом запуске пройди логин в браузере.

03 ▸ Скорми Claude промпт Карпати

Отправь Claude два промпта подряд. Первый — «обёртка» с просьбой создать каркас. Второй — полный промпт Карпати с описанием архитектуры.

Промпт 1 — обёртка

prompt 1 / 2 Привет! Я хочу, чтобы ты создал структуру папок по образцу, который я тебе сейчас скину. Если есть вопросы — задавай, но в целом просто следуй инструкции и создай каркас моей базы знаний. [вставь ниже полный текст Промпта 2]

Промпт 2 — архитектура LLM wiki

Полный промпт Карпати на русском — в разделе 📎 Доп материалы, а также оригинал в его Gist. Ключевые блоки промпта:

  • Три слоя памяти: raw sources → wiki (markdown) → schema (CLAUDE.md).
  • Четыре операции: Ingest, Query, Lint, Maintenance.
  • Эффект масштабирования: один источник затрагивает 10–15 страниц, Claude не устаёт их обновлять.

Через 2–3 минуты Claude создаст каркас:

Jarvis/
├── CLAUDE.md
├── raw/
│   ├── sources/
│   ├── web-clipped/
│   └── assets/
└── wiki/
    ├── entities/
    ├── concepts/
    ├── sources/
    ├── synthesis/
    ├── index.md
    └── log.md

04 ▸ Запусти Jarvis первой статьёй

Дай Claude задачу — проскрапить статью и разложить её по wiki. Пример — разбор статьи с Хабра про 15 идей заработка в ИИ.

Промпт 3 — первый ingest

prompt 3 / 4 Чувак, хочу сделать wiki по заработку на нейросетях и AI-автоматизации. Назовём Jarvis — мой персональный ИИ-ассистент, который знает меня и помогает принимать лучшие решения про то, как зарабатывать на ИИ. Я буду кидать тебе статьи, кейсы, идеи, инструменты — и хочу, чтобы ты интегрировал всё это в структуру. Сначала — вот первый URL. Пройди по нему, скрапь весь контент (текст и картинки), разложи по raw/sources/ и сделай первую wiki-страницу в wiki/. URL: https://habr.com/ru/articles/990854/

Claude скачает текст и картинки, создаст страницы сущностей (Otli, Mogilko, Martel), концептов («value ladder», «retainer model», «awareness arbitrage»), обновит index.md и log.md.

05 ▸ Установи Obsidian Web Clipper

Расширение сохраняет любую веб-страницу одной кнопкой прямо в твой vault — Claude потом её обработает.

  1. Поставь расширение из Chrome Web Store.
  2. Кликни на иконку пазла 🧩 справа от адресной строки → закрепи Obsidian Web Clipper 📌.
  3. Открой любую страницу → клик на фиолетовый ромб → шестерёнка ⚙ снизу → Settings.
  4. В разделе Vaults введи имя своего vault точно как в Obsidian → Enter.
  5. В разделе Templates → New template:
    • Name: Jarvis
    • Note location: raw/web-clipped/
    • Default storage vault: имя твоего vault

06 ▸ Подключи папку как vault в Obsidian

  1. Вернись в Obsidian → внизу слева клик на имя vault → Open folder as vault.
  2. Выбери папку Jarvis → Open → Trust author если спросит.
  3. В левой панели увидишь: CLAUDE.md, README.md, raw/, wiki/.
  4. Включи граф знаний: в левой панели иконка «круг с точками» → клик.
  5. Потягай узлы — видно сеть связей между сущностями, концептами и источниками.

07 ▸ Задай Jarvis первый вопрос

Проверь, что система работает как синтезатор, а не как простой поиск. Claude должен сослаться на файлы, которые сам же и создал.

Промпт 4 — составной вопрос

prompt 4 / 4 Окей, проверим систему. На основе всего что ты про меня знаешь — профиль, wiki, концепты, статья с Хабра — ответь: я хочу в ближайшие 3 месяца вырасти на YouTube и параллельно начать продавать услуги студии через канал. Какие 3 конкретные стратегии из того что мы уже собрали в Jarvis я могу применить? Предложи сочетания — концепт плюс автор равно конкретное действие. И перечисли какие файлы из wiki ты использовал.

Claude создаст новый файл в wiki/synthesis/. На вопрос «какая стратегия сработала?» через месяц он обратится к этому же файлу — это и есть compound-эффект Jarvis.

08 ▸ Пойми ограничения системы

У Obsidian-RAG есть жёсткий потолок — 99% блогеров про него молчат.

  • Индекс растёт линейно — запросы дорожают с каждым новым источником.
  • Нет семантического поиска — Claude ищет по ключам, а не по смыслу.
  • Не справляется с архивами 10 000+ файлов — упрётся в контекст.

Формула «идеальной памяти» из четырёх слоёв

Слой 1 · бейдж
CLAUDE.md — личность и правила. Меняется редко.
Слой 2 · мастерская
Obsidian — активное мышление, текущие проекты, идеи, связи.
Слой 3 · склад
Pinecone — архив транскриптов и больших статичных данных с векторным поиском.
Слой 4 · ресёрч
NotebookLM — глубокое исследование на 200+ источниках одновременно.

Три слоя архитектуры из промпта Карпати

  1. Raw sources — неизменная коллекция: статьи, PDF, картинки, данные.
  2. Wiki — генерируемые markdown-файлы: summaries, страницы сущностей, концепты, сравнения, синтез.
  3. Schema — CLAUDE.md: структура, соглашения, workflows.

Четыре операции Jarvis

  • Ingest — новый источник → прочитать → обсудить → summary → обновить индекс, сущности, концепты, лог (1 источник = 10–15 страниц).
  • Query — вопрос → найти релевантные страницы → прочитать → синтезировать с цитатами → сохранить в wiki/synthesis/.
  • Lint — health-check: противоречия, устаревшие факты, страницы-сироты, недостающие концепты, битые ссылки.
  • Maintenance — Claude обновляет файлы, человек курирует источники и задаёт вопросы.

🔑 Ключевые мысли

  1. Claude Code не устаёт поддерживать wiki. Люди бросают личные базы, потому что maintenance растёт быстрее ценности. Claude касается 15 файлов за один проход — система остаётся живой.
  2. Один источник → десятки связей. Ingest обновляет не только summary, но и страницы сущностей, концептов, индекс и лог. Compound-эффект на каждом новом источнике.
  3. Obsidian — это мастерская, а не архив. Для 10k+ файлов нужен Pinecone. Для глубокого ресёрча — NotebookLM. Jarvis — только активное мышление и проекты.
  4. Плоские Markdown-файлы — ключ. Claude легко читает, редактирует и связывает их. Ты видишь всё глазами в Obsidian. Формат побеждает любую хитрую БД для активной работы.
  5. CLAUDE.md — это закон системы. Там описаны структура, правила именования, workflow. Claude сверяется с ним при каждом действии и сам не ломает инвариант.
