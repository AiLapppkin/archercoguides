Превратить Claude Code в команду из 100 агентов через Ruflo

💡 О чём этот гайд

Ruflo (бывший Claude Flow, переименован в январе 2026) — это платформа оркестрации поверх Claude Code. Запускает рои из 100+ специализированных агентов, которые координируются через SendMessage, шарят память и учатся. Под капотом — WASM-ядра на Rust для policy engine, эмбеддингов и системы доказательств. Одна команда init — и Claude Code получает «нервную систему».

✅ Что ты освоишь

  1. Поймёшь архитектуру Ruflo: Router → Swarm → Agents → Memory с обратной петлёй обучения
  2. Установишь Ruflo как плагин Claude Code или MCP-сервер одной командой
  3. Подключишь общую память AgentDB с HNSW-индексированием
  4. Используешь федерацию для безопасного обмена агентами между машинами
  5. Запустишь Web UI на flo.ruv.io или Goal Planner на goal.ruv.io
  6. Соберёшь первый swarm под повторяющуюся задачу в своём бизнесе

1. Понять, чем рой отличается от одного агента

Claude Code — один умный воркер. Умный, но один.

Ruflo — менеджер, который нанимает команду Claude-воркеров, раздаёт каждому задачу и следит, чтобы они обменивались заметками.

Состав команды

Внутри 100+ специализированных ролей: coder, tester, reviewer, architect, security, backend-dev, mobile-dev, ml-developer, cicd-engineer и другие. Делятся на queens (принимают большие решения) и workers (делают чёрную работу). Свои роли пишутся в простом YAML-файле.

Старая модель: даёшь Claude один большой промпт и молишься. С Ruflo — говоришь, что нужно, а платформа режет задачу на куски. Архитектор планирует. Кодер пишет. Тестер проверяет. Ревьювер чинит. Все шарят одну память и не забывают, что было решено пять минут назад.

2. Понять архитектуру Ruflo

Самообучающаяся архитектура с обратной петлёй:

User → Ruflo (CLI/MCP) → Router → Swarm → Agents → Memory → LLM Providers
                       ^                                |
                       +────── Learning Loop ←──────────+

Каждая выполненная задача возвращается в Memory, обогащает паттерны, и следующий заход выполняется быстрее. Под капотом WASM-ядра на Rust обеспечивают изоляцию и скорость для policy engine, эмбеддингов и проверки доказательств.

3. Установить Ruflo как плагин Claude Code

Это рекомендуемый способ. Добавляет skills, commands, agents и MCP-тулы напрямую в Claude Code:

# Подключи маркетплейс
/plugin marketplace add ruvnet/ruflo

# Поставь ядро
/plugin install ruflo-core@ruflo

# Добавь нужные модули
/plugin install ruflo-swarm@ruflo
/plugin install ruflo-autopilot@ruflo
/plugin install ruflo-federation@ruflo

Этого хватит. Внутри Claude Code появляется AI-команда, и ты продолжаешь работать в привычном интерфейсе. Маленькие hooks следят за тем, что ты делаешь, и роутят работу нужному агенту без явных команд.

4. Альтернатива: визард или MCP-сервер

Если плагин-система не подходит — есть три варианта:

# Через визард (выбираешь роли, память, провайдера)
npx ruflo@latest init --wizard

# Глобальная установка
npm install -g ruflo@latest

# Один скрипт-инсталлятор
curl -fsSL https://cdn.jsdelivr.net/gh/ruvnet/ruflo@main/scripts/install.sh | bash

# Как MCP-сервер
claude mcp add ruflo -- npx -y @claude-flow/cli@latest

314 MCP-инструментов из коробки

В версии 3.6.12 количество нативных тулов выросло с 87 до 314 — это в 3.5 раза больше. CI-проверки, security-сканы, облачные деплои, observability. Тебе не нужно знать каждый — нужный тул вызовется сам через hooks.

5. Подключить общую память AgentDB

Память — самая важная часть. Ruflo использует AgentDB с HNSW-индексированием — векторное хранилище для AI-агентов. Авторы заявляют, что оно в 150–12 500 раз быстрее базового поиска.

Что это даёт на практике:

  • Агенты помнят всё между сессиями
  • Помнят между проектами
  • Помнят между машинами
  • Закрыл ноут — вернулся завтра — команда продолжает с того же места

Сверху работают SONA (нейронные паттерны) и ReasoningBank — система анализирует трассы успешных решений и улучшает маршрутизацию задач.

6. Использовать федерацию между командами

Federation — главная фича апдейта. Это «Slack для агентов»: агенты на разных машинах, в разных орг-аккаунтах или облачных регионах находят друг друга, проверяют идентичность и обмениваются работой.

Как устроена безопасность

Под капотом — mTLS и ключи ED25519. Удалённые агенты стартуют как недоверенные. Идентичность доказывается через challenge-response. Никаких API-ключей или общих секретов.

Плюс встроенная PII-pipeline из 14 типов сканирует каждое исходящее сообщение и применяет политики: BLOCK, REDACT, HASH или PASS в зависимости от уровня доверия.

Доверие считается по формуле 0.4×success + 0.2×uptime + 0.2×threat + 0.2×integrity. Апгрейды требуют истории. Даунгрейды — мгновенные. Реальный пример: банк A и банк B обмениваются сигналами о фроде, не передавая данные клиентов. Раньше это были месяцы юридической переписки. Сейчас — две команды.

Compliance готов из коробки: HIPAA, SOC2, GDPR audit-trails. Каждое событие пишется в журнал и ищется через HNSW.

7. Поставить плагины из маркетплейса

В Ruflo 32 нативных плагина для Claude Code и 21 npm-плагин. Ставятся как приложения на телефоне.

Главные плагины

  • ruflo-core — основа, health checks, plugin discovery
  • ruflo-swarm — координация команды агентов
  • ruflo-autopilot — агенты работают сами в цикле
  • ruflo-loop-workers — фоновые задачи по таймеру
  • ruflo-federation — кросс-машинная коллаборация
  • ruflo-agentdb — векторная база для памяти
  • ruflo-rag-memory — гибридный поиск, граф-обходы, ранжирование
  • ruflo-aidefence — блокирует prompt injection, детектит PII

С ruflo-loop-workers агент может проверять входящие раз в час, сортировать лиды, готовить черновики ответов — и к утру всё уже на столе.

8. Запустить Web UI на flo.ruv.io

У Ruflo есть полноценный Web UI с MCP tool calling — мульти-модельный чат, в котором тулы ядра вызываются прямо из разговора.

Доступные модели по умолчанию

  • Qwen 3.6 Max (default)
  • Claude Sonnet 4.6
  • Claude Haiku 4.5
  • Gemini 2.5 Pro / Flash
  • OpenAI через OpenRouter
  • Любой OpenAI-совместимый эндпоинт: vLLM, Ollama, LM Studio, Together, Groq

~210 тулов в 5 группах (Core, Intelligence, Agents, Memory, DevTools) плюс 18-tool gallery, которая работает прямо в браузере через WASM. Один ответ модели может запустить 4–6 тулов параллельно. Без аккаунта, без API-ключа: открыл flo.ruv.io — попробовал. Свой инстанс деплоится через Docker.

9. Описать цели на простом английском через Goal Planner

Goal Planner на goal.ruv.io — это GOAP A* планировщик (Goal-Oriented Action Planning). Опиши цель словами — Ruflo разложит её на предусловия, действия и найдёт кратчайший путь через пространство состояний.

Пример: пишешь «ship the auth refactor with tests and a PR» — Ruflo вытаскивает критерии успеха, ограничения, неявные предусловия. Никакого JSON, никакого DSL.

Дашборд goal.ruv.io/agents показывает каждого запущенного агента: роль, текущий шаг, namespace памяти, бюджет токенов, статус. Можно убить раннавей-воркера или переназначить задачу.

10. Запустить офлайн через Ollama

Ruflo работает офлайн. Подключаешь локальную модель через Ollama (или плагин ruflo-ruvllm) — и приватные данные не покидают компьютер.

Для юристов, врачей, бухгалтеров и всех, у кого чувствительная информация, это меняет правила игры. AI-команда есть, приватность сохранена. Поверх — система SONA, которая учится на твоих трассах через MicroLoRA-адаптеры.

11. Собрать первый swarm под реальную задачу

Маркетинговое агентство

Один агент тянет данные. Второй пишет первый драфт отчёта. Третий проверяет цифры. Четвёртый форматирует финальный документ. Работают параллельно, пока ты занят другим.

Риелтор

Swarm следит за новыми листингами, пишет описания объектов, готовит письма для базы покупателей и сохраняет всё в память — следующий заход будет быстрее.

Интернет-магазин

Writer-агент, SEO-агент, product-researcher и image-describer. На каждый новый товар swarm пишет описание, подбирает ключевики, готовит социальный пост. Ты только аппрувишь.

Коуч

Один агент следит за календарём, второй готовит фоллоу-апы, третий конспектирует звонки из заметок, четвёртый обновляет CRM. Работают, пока ты спишь.

12. Понять цифры производительности

На бенчмарке SWE-bench Ruflo решает 84.8% задач — выше многих платных агентских инструментов.

API-вызовы режет на ~75%: лёгкая работа уходит на дешёвые быстрые модели, тяжёлая — на большую модель. Платишь только за то, что реально требует мозгов.

Проект уже не маленький эксперимент: 41 800+ звёзд на GitHub, ~500 000 загрузок, ~100 000 пользователей в 80+ странах. 6 000+ коммитов, 250 000 строк кода на TypeScript и Rust.

13. Учесть подводные камни

Что нужно знать заранее

  1. Терминал. Простые команды, не код. Если чёрный экран пугает — посмотри 10-минутное видео и норм.
  2. Чёткая задача. Если не знаешь, чего хочешь — рой агентов это не починит. Garbage in, garbage out.
  3. Claude Pro / Max. В апреле 2026 поменялись правила работы сторонних тулов с подпиской. Команда Ruflo держит совместимость, но проверь актуальность.
  4. NPM-имя. Часть команд (например MCP-добавление) до сих пор использует @claude-flow/cli — это исторический пакет, бренд переименован, npm-имя осталось.

14. Запустить план на 30 дней

  1. Не пробовал Claude Code — начни с него. Бесплатно.
  2. Подключи Ruflo через /plugin marketplace add ruvnet/ruflo
  3. Запусти npx ruflo@latest init --wizard с дефолтами
  4. Возьми одну повторяющуюся задачу в бизнесе: отчёты, письма, фоллоу-апы, контент-драфты
  5. Собери под неё маленький swarm
  6. Через неделю — добавь следующую задачу
  7. За месяц кусок работы переедет на автопилот
